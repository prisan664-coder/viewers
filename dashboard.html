1. Earning & Reward Conditions

To control payouts against overpaying and motivate continuous active user engagements:

  • Accumulation Rate: Earnings calculated on real watch time at 1.38 KIP per second (approx. 5,000 KIP per hour).
  • Minimum Withdrawal Threshold: Fixed at 4,000,000 KIP per request to filter out short-term exploiters or bots.
  • Video Re-watch Restriction: Each video item within the playlist generates earnings at a maximum of 1 time per day per account (System resets every midnight) to avoid overnight repeat loop runs.
  • Fee Deductions: Every withdrawal request is subject to a transaction/blockchain network fee deduction (e.g., 2% - 5% or a flat fee determined by the Admin).
2. System Security Conditions

To protect the core framework against balance manipulation and code exploits:

  • Session Monitor System: One user account can log in and stream video on only 1 screen/1 device simultaneously. Dual session overlaps will trigger an instant disconnection.
  • Server-Side Verification: Accumulated balance calculation must match the user browser time with the core master database. Front-end edits (Inspect Element) will be ignored; the platform relies strictly on backend values.
  • Anti-Bot Captcha Check: Randomly prompts identity verification buttons or basic questions every 1-2 hours to ensure active manual monitoring rather than automated script clicking.
3. Internet & Connection Conditions

Prevents balance disputes arising from user connection disconnections:

  • Signal Stability: Rewards accumulate strictly when videos are streamed continuously showing the "Now Streaming" status. Internet drops, signal failures, or video pausing will instantly halt the accumulation.
  • VPN / Proxy Prohibition: Users must connect via home broadbands or standard mobile data. Usage of IP manipulators, VPNs, or Proxies is strictly banned. Anomalous IPs will cause a temporary account suspension for review.
4. Strict Prohibitions

Violations grant administrators absolute rights to terminate accounts and forfeit rewards instantly without notice:

  • Multi-tabbing Banned: Opening multiple dashboards at the same time to double earning speeds is prohibited.
  • No Automation Scripts: Usage of bots, hacker software, macros, or auto-clickers of any form is strictly forbidden.
  • Multi-Accounts / Sybil Attack: Strictly limited to 1 user profile per individual. If discovered managing multiple clone profiles for distributed payouts, all associated wallets will face termination.
5. User Safety & KYC

Establishes site credibility and defines liability parameters regarding data entries:

  • KYC Identity Verification: Users must provide authentic matching legal data (Full Name, Date of Birth, Phone, Bank details, Binance link) to prevent money laundering and identity theft.
  • Data Privacy: Information submitted via withdrawal forms remains strictly confidential within the secure admin system, used solely for payouts, and never shared with third parties.
  • Data Accuracy Liability: If a user submits incorrect bank details or Binance Wallet Links causing transfer failures, the system accepts zero liability for losses. Verification prior to submission is mandatory.
6. Pending Verification Period
  • Review Period: Upon withdrawal submission, balances do not process instantly. They enter a "Pending" status for 24-72 hours allowing admins to review historical watch logs against exploits.
  • Right of Refusal: Admins hold absolute authority to cancel malicious requests, return balances for audit, or freeze active accounts if operational anomalies are discovered in logs.
7. Daily Earning Cap
  • Daily Revenue Limits: Standard accounts possess a maximum daily earning limit (e.g., up to 120,000 KIP or capped at 24 watch-hours daily) preventing bots from running infinitely.
8. Demotion & Expiration Conditions
  • Inactive Accounts: Profiles failing to log in or watch videos for 30 consecutive days are flagged inactive. Accumulated balances reset to 0 or accounts face closure to reduce database overheads.
  • Reward Adjustment Rights: Administrators reserve absolute rights to modify or scale back reward rates (KIP/hour) or raise withdrawal thresholds based on market economies without prior notification.
9. Limitation of Liability
  • Technical Outages: Under force majeure (server failures, database corruption, global network disasters) resulting in potential data loss, the platform commits to recovery efforts but denies liability for compensations beyond verifiable log evidence.
